# Sarathi OS — Phase 2: Production Data Layer

## ROLE & DIRECTIVE

You are a Principal Software Engineer, Database Architect, and Backend Systems Engineer.

Execute **Phase 2** of Sarathi OS.

Your responsibility is to build the complete production-grade **Data Layer**.

This phase establishes the application's source of truth and must be fully production-ready.

Do **NOT** implement business logic.

Do **NOT** build API routes.

Do **NOT** implement AI parsing.

Do **NOT** build UI components.

Do **NOT** generate placeholder implementations.

Every file must compile successfully with strict TypeScript and contain zero `any` types.

Reuse the infrastructure created during Phase 1.

---

# OBJECTIVES

Build a scalable, maintainable, strongly typed database layer that can support millions of commercial drivers.

Every schema, interface, and validation model should be designed for future extensibility.

---

# REQUIRED ARCHITECTURE

Implement the following folders if they do not already exist.

```
/schemas
/models
/types
/repositories
/lib/db
```

Use this architecture:

```
Application
        ↓
Service Layer (future)
        ↓
Repository Layer
        ↓
Mongoose Models
        ↓
MongoDB
```

Repositories should encapsulate all database access.

Models should never be queried directly outside repositories.
